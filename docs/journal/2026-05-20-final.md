# 2026-05-20 — final overnight session entry

## v3 midsize training — the angle finally shows

After fixing the two corpus bugs (over-aggressive `<eos>` packing, hardcoded SQuAD prefix) and adding source-weighting to the dataloader (Gutenberg 8×, SQuAD 4×, Common Pile 1×), the v3 midsize 50M model trained on 200M tokens produces qualitatively different output from v1 and v2.

### Loss trajectory comparison

| Run | Params | Tokens | Corpus | Final loss | Sampling style |
|---|---|---|---|---|---|
| debug-10m | 12 M | 50 M | uniform | 4.0 | academic continuation |
| midsize-50m-v1 | 49.6 M | 200 M | uniform, `<eos>` bug | 3.1 | empty (model emits `<eos>` after `<reply>`) |
| midsize-50m-v2 | 49.6 M | 200 M | uniform, eos fix, SQuAD prefix bug | 3.2 | SQuAD-style "Here's some context:" boilerplate |
| midsize-50m-v3 | 49.6 M | 200 M | weighted (gut 8×, sq 4×, cp 1×), clean SQuAD | 3.2 | **conversational dialogue** |

The output is incoherent at the *fact* level (50M model on 200M tokens is way undertrained for coherent reasoning), but the *register* is unmistakable: 19th-century novel dialogue, no AI-assistant boilerplate, no "Great question!" preamble, no "As an AI" disclaimers.

## What this means for v1.0

The 50M / 200M-token result is **proof of concept**, not the final v1.0 model. What it demonstrates:

1. The angle (no synthetic assistant data, no SFT → no sycophancy) **works** even at this tiny scale.
2. The corpus pipeline produces a model whose register matches the training data, not the default-RLHF register.
3. Source weighting matters more than raw token count when the source distribution is skewed.

For v1.0 (350M, 7B+ tokens) the same recipe should produce a model that's:
- Still sycophancy-free (the angle scales)
- Actually coherent (Chinchilla-optimal training fixes the incoherence)
- Capable of multi-turn recall (long-conv eval should improve at scale)
